## Prerequisites

    **Podman on FreeBSD currently requires root.** All commands must be run as root (or via `doas`/`sudo`).

Before deploying, ensure your host environment is ready. See the [Quick Start Guide](../quick-start.md) for host setup instructions.


## Deployment


=== ":material-docker: Podman Compose"

    ```yaml
    services:
      home-assistant:
        image: @REGISTRY@/home-assistant:latest
        container_name: home-assistant
        environment:
          - PUID=@PUID@
          - PGID=@PGID@
          - TZ=@TZ@
        volumes:
          - "@CONTAINER_CONFIG_ROOT@/@HOME_ASSISTANT_CONFIG_PATH@:/config"
        restart: unless-stopped
    ```


=== ":material-console: Podman CLI"

    ```bash
    podman run -d --name home-assistant \
      -e PUID=@PUID@ \
      -e PGID=@PGID@ \
      -e TZ=@TZ@ \
      -v @CONTAINER_CONFIG_ROOT@/@HOME_ASSISTANT_CONFIG_PATH@:/config \
      @REGISTRY@/home-assistant:latest
    ```

=== ":simple-ansible: Ansible"

    ```yaml
    - name: Deploy home-assistant
      containers.podman.podman_container:
        name: home-assistant
        image: @REGISTRY@/home-assistant:latest
        state: started
        restart_policy: always
        env:
          PUID: "@PUID@"
          PGID: "@PGID@"
          TZ: "@TZ@"
        volumes:
          - "@CONTAINER_CONFIG_ROOT@/@HOME_ASSISTANT_CONFIG_PATH@:/config"
    ```
